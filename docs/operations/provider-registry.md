This registry defines operational expectations, not guaranteed provider behavior. Verify upstream documentation and free-tier terms before each release. A provider is independently degradable: failure of one source must not erase freshness or provenance for another.

## Provider state contract

Every adapter reports `connecting`, `healthy`, `reconciling`, `stale`, or `failed`, plus last event, last heartbeat, reconnect count, sequence-gap count, and a redacted last error. Every derived response includes provider, source kind (`exchange-native`, `modeled`, or `fallback`), observed/exchange timestamp, ingest/publish timestamp where available, coverage, freshness, and schema version.

No source is silently substituted. Observed liquidation prints are never merged into modeled bands without separate provenance. Native trade CVD and candle-proxy CVD use distinct source labels. Missing option strikes/maturities remain masked rather than interpolated. A free-tier quota breach is an explicit capacity incident.

## Review cadence

Review quotas and documented cadence monthly and before production rollout. Record changes in release notes, update alert thresholds, and test the affected adapter fixture. Production credentials are not required for the public sources in this registry.
